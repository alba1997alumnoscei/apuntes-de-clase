Media Queries

Qué son las MQ?

Declaración en CSS que nos permite definir cómo se va a visualizar nuestra web. En CSS vamos a hacer algo que nos dice cómo se va a ver nuestra web.

sintaxis de un MQ

una MQ se compone de múltiples partes:

  • Keyword: @media es una palabra clave reservada
  • MediaType: un tipo de medida. Ej: screen, print, all
  • Operador/es: Ej: and, not, only. Nota: si no uso un breakpoint, no es necesario usar un operador
  • Breakpoints: condición a cumplirse. Ej: max-width, min-width
  • Reglas CSS: bloque de reglas
orden: /*keyword MediaType Operador/es (breakpoints) {reglas de CSS}*/

Tipos de Media (screen/print)

- Print: destinado a material de vista de impresión
- Screen: destinado a pantallas a color (utilizado por defecto)
- All: Todos los dispositivos
- Pueden encontrar otros tipos que ya no usan: tv, handheld, speech, braile, etc

Tips para utilizar media Queries

  1. los MQ sobreescriben las reglas de CSS anteriores, PERO no poseen mayor especificidad que las otras reglas, por eso debemos agregarlas al final del documento.
  2. deben utilizar los mismos selectores que quieran reemplazar
  3. Pensar bien los breakpoints
  4. Podemos utilizar una @media para cada selector, o una para todos los selectores (la más recomendada). Es decir, agrupar todas las reglas para mobile, todas las reglas para tablets, y todas las reglas para desktop

Mobile vs. Tablet vs. Desktop

- Mobile (55%): tamaño de pantalla 320px-768px
- Tablet (2%): tamaño de pantalla 768px-1024px
- Desktop (43%): tamaño de pantalla 1024px-1920px o más

Mobile first: Planificar nuestra web para dispositivo movil y después hacer los MQ para tablet o desktop. Motivos: es el más usado, el usuario tiene que ver la información de mínimo (cuál es lo minimo que tiene que ver?), por ejemplo la publicidad no quiero enseñarsela en el movil y solo en tablet o desktop.

uso de MQ en HTML

Podemos utilizar los @media directamente en el HTML utlizando el atributo "media". Antes que nada, debemos asegurarnos de que nuestro HTML incluya la etiqueta META en el head. meta name="viewport" content="width=device-width, initial-scale=1.0"
  • etiquetas de Link link rel="stylesheet" href=".css/tablet.css" media="screen and (min-width: 768px)" le estoy diciendo a mi etiqueta del link que va a funcionar siempre y cuando la pantalla tengo una anchura minima de 768px.
  • Etiqueta en style (no recomendada): style media="screen and (min-width: 768px)
  • Etiqueta de picture:
    /*esta es la que se muestra en pantalla*/
